# V1-4 更新总结

- 更新时间：2026-04-03 14:22（Asia/Hong_Kong）
- 覆盖范围：`V1` 至 `V4.4b`
- 文档定位：用于系统总结当前课题在数据处理、主表构建、建模主线、关键结论、方法边界与后续方向上的阶段性认识

## 1. 当前总结的核心结论

截至目前，课题已经完成了从原始 SYR4 数据可拼接性审计，到第二层与第三层主表构建，再到第三层 `PWS-year` 主线的第一轮正式机制增强实验。当前最稳妥的阶段性判断是：

1. `PWS-year` 适合作为全国尺度风险识别主表，但不适合作为精细机理过程表。
2. `facility-month` 更适合作为机制分析、专题 reduced dataset 和后续高信息子样本分析层级。
3. 在第三层主线上，`TOC` 是目前最明确、最可信的增量变量。
4. `free_chlorine` 在第三层主线上只有弱边际增益，且当前无法稳健拆分其数值信号、缺失模式信号和样本选择效应。
5. `total_chlorine` 当前不具备继续进入第三层正式增量实验的条件。
6. 当前最强的第三层主线结论不是“找到最优模型”，而是“已经探明哪些变量可以支撑全国主线，哪些变量不适合继续在第三层硬推”。

## 2. 从 V1 到 V4.4b 的工作链条

### 2.1 V1：严格样本级保守基线

`V1` 的定位不是主建模，而是：

- 做严格样本级可拼接性审计
- 做第一层 sanity check
- 确认核心变量在原始 SYR4 中到底能否高质量对齐

这一轮的关键结论是：

- 严格样本级下，`TTHM + pH + alkalinity + TOC + free_chlorine` 的完整样本几乎不存在
- 严格四键对齐无法直接支撑后续多变量主建模
- 因此课题不能把“最严格对齐”误当成“最适合作为主分析底表”

这一步是合理的，而且非常重要，因为它避免了后续在不现实的数据基础上强行建模。

### 2.2 V2：三层级数据基础审计

`V2` 的作用是把数据问题从“直觉”变成“有证据的分层判断”：

- 第一层：严格样本级
- 第二层：`facility-month`
- 第三层：`PWS-year`

这一轮把研究设计拉回到合理轨道上：

- 第一层适合保守审计，不适合作为主建模底表
- 第二层适合机制问题和高信息专题分析
- 第三层适合全国尺度风险识别和系统级主模型

到这里，课题已经明确：机器学习主线与机制解释主线不能再使用同一底表硬做。

### 2.3 V3 / V3.5：主表与建模输入层构建

`V3` 和 `V3.5` 的核心产出是：

- `V3_facility_month_master`
- `V3_pws_year_master`
- `V4_pws_year_ml_ready`

这一步的合理性在于：

- 把极碎、极不均匀的原始 SYR4 观测表转成可审计、可复用的数据产品
- 明确字段来源、聚合规则、缺失语义、禁止误用规则
- 让后续实验不再建立在临时口径上

但这一步也带来一个必须持续记住的代价：

- 从原始样本到 `PWS-year` 的聚合，意味着模型学习的已经不是“微观形成过程”，而是“系统-年份风险画像”

因此，第三层上的任何结果都不应直接等同于环境过程机制结论。

### 2.4 V4：第三层主线正式实验

第三层主线已经走过以下路径：

1. `baseline`
2. `baseline + pH + alkalinity + missing flags`
3. `+ TOC + toc_missing_flag`
4. `+ free_chlorine + free_chlorine_missing_flag`
5. `total_chlorine readiness audit`

这条链条本身是当前课题方法上最稳的一部分，因为它具备：

- 统一输入表
- 统一切分
- 统一模型
- 统一评价指标
- 明确的对照链

也正因此，当前结论具有可比性，不是不同口径下随意拼出来的结果。

## 3. 到目前为止，哪些结论最稳

### 3.1 最稳的变量增量结论：`TOC`

当前第三层主线上，`TOC` 是最明确的增量变量。

主要证据包括：

- 在 `anchored` 任务上，`TOC` 带来明显且稳定的边际增益
- 在 `regulatory` 任务上，`TOC` 带来部分稳定增益，但不是所有指标都持续提升
- `TOC` 的 complete-case 版本仍然可训练，而且结果明显改善

因此，`TOC` 不是“看起来有用”，而是当前第三层主线上最可信的机制增强变量。

### 3.2 最稳的主线任务结论：`anchored` 更容易学到清晰信号

到目前为止，`anchored` 任务整体比 `regulatory` 更容易获得清晰、稳定的提升。

原因不是它更“真实”，而是它通过 `<=40` 与 `>=80` 的端点锚定，放大了高低风险的差异结构，使模型更容易识别。

这意味着：

- `anchored` 更适合做机制信号放大镜
- `regulatory` 更适合作为贴近监管阈值的正式任务

后续不能因为 `anchored` 分数更高，就把它直接等同于整体风险预测主线。

### 3.3 当前最需要谨慎的变量：氯残留变量

`free_chlorine` 和 `total_chlorine` 的实验共同说明：

- 这类变量在第三层 `PWS-year` 上覆盖率极低
- 即使有微弱增益，也很难证明增益主要来自数值本身
- 缺失模式和样本选择效应很可能是重要来源

所以当前最稳妥的结论不是“氯残留不重要”，而是：

- 氯残留变量与第三层年尺度主表的匹配度较差
- 在当前口径下，不适合继续把它们作为第三层主线待增量变量

## 4. 当前流程中有哪些合理之处

### 4.1 从数据可拼接性开始，而不是直接建模

这是整个课题当前最合理的地方之一。

很多项目会跳过“数据是否能支持这个问题”的检验，直接训练模型。当前项目先做 `V1` 和 `V2`，实际上避免了很多后续方法错误。

### 4.2 分层设计是正确的

当前已经把不同层级的用途分开：

- 第一层：严格审计
- 第二层：机制专题
- 第三层：全国主线

这是和 SYR4 实际数据结构最一致的设计之一。

### 4.3 逐步增强实验链是合理的

当前不是一次性把所有变量都塞进模型，而是：

- 先 baseline
- 再机制核心
- 再增量变量
- 再做 complete-case 和可执行性审计

这种做法保证了每一步结论都有明确参照物。

### 4.4 持续保留解释边界是正确的

当前流程一直在强调：

- 缺失模式不等于环境机制
- `第二级样本` 不等于全国样本
- complete-case 不等于更真实，只是另一种对照
- 分数提升不等于因果发现

这些判断使当前项目在方法上仍然处于一个可控的状态。

## 5. 当前流程中最值得警惕的不合理处

### 5.1 第三层主线容易被误读成机制模型

`PWS-year` 是年度聚合后的系统级表，它非常适合做：

- 风险识别
- 样本分层
- 全国尺度比较

但它不适合被写成：

- 细粒度环境过程模型
- 真实工艺控制机制模型
- 因果过程解释模型

如果后续写作不够克制，第三层上的统计相关性很容易被过度解释。

### 5.2 `第二级样本` 增量结论受样本选择影响

`第二级样本` 是高信息样本，不是全国随机样本。变量越往后加，越有可能只在“本来更容易被测、信息更全”的系统上起作用。

因此：

- `第二级样本` 上的结果可以作为增量实验结论
- 但不应自动上升为全国主线最终结论

### 5.3 高风险分类任务会损失连续暴露信息

分类任务很适合当前阶段，因为它便于对接监管阈值，也更容易形成清晰的实验链。

但代价是：

- 中间梯度信息被压缩
- `anchored` 任务丢掉了灰区样本
- 模型对极端端点的学习，未必能完全代表一般风险变化

因此当前分类主线合理，但后续若要进一步深化机制理解，不能永远只停留在阈值分类层面。

### 5.4 缺失模式是当前最大的统计风险

当前项目虽然已经比一般做法更谨慎，但仍需明确：

- SYR4 里的缺失很可能不是随机缺失
- 是否有某个变量的观测，本身就反映了监测制度与系统特征
- 当变量极其稀疏时，模型可能主要在学“谁被测了”，而不是“值是多少”

这在 `free_chlorine` 和 `total_chlorine` 上已经被非常清楚地暴露出来。

### 5.5 变量顺序会影响“谁被看作重要”

当前使用的是逐步增强链条，因此：

- 后加变量的价值是在前面变量已进入模型后测得的
- 如果变量间高度相关，后加变量可能被低估

因此当前结论更准确的表达应该是：

- “在当前既定实验路径下，`TOC` 是最强增量变量”

而不是：

- “在所有可能模型中，`TOC` 绝对是最重要变量”

## 6. 当前最好用版本应如何理解

当前“最好用”的版本不能只看单一分数，而要分三种含义：

### 6.1 如果看纯性能上限

当前分数最高的是：

- `V4.3`
- `tthm_anchored_risk_prediction`
- `mechanistic_stage2_toc_increment_complete_case`

这个版本代表：

- 在高信息 complete-case 子集上，`TOC` 的机制信号非常强

但它不适合作为全国主线正式版本，因为样本规模明显缩小。

### 6.2 如果看当前正式主线可用版本

当前最适合作为正式主线结果的是：

- `V4.4`
- `tthm_anchored_risk_prediction`
- `mechanistic_stage3_free_chlorine_increment_v4_4_1`

原因不是它一定最有解释力，而是：

- 它是在 full `第二级样本` 上当前表现最好的完整版本
- 它保留了当前主线对照结构

### 6.3 如果看最可信的机制增量结论

当前最可信的不是 `free_chlorine`，而是：

- `baseline + pH + alkalinity + TOC`

也就是说，当前项目最重要的结论不是“我们找到了终极最优模型”，而是：

- `TOC` 在第三层主线上已经被相对稳健地确认为有价值
- 氯残留变量则尚未获得同等强度的证据

## 7. 与实际环境意义的冲突点

### 7.1 氯残留变量在第三层上的环境意义弱化

氯残留本身是更偏运行状态的变量，时间波动快、场景依赖强。

把它压缩为 `PWS-year` 年均值后，再叠加极高缺失率，就会导致：

- 环境意义被弱化
- 统计信号更可能由缺失模式和样本选择主导

这解释了为什么 `free_chlorine` 有微弱增益但难以解释，`total_chlorine` 更几乎无法继续推进。

### 7.2 第三层并不能替代第二层

如果继续试图在第三层上解决越来越细的机制问题，就会和变量自身的环境意义发生冲突。

当前结果实际上已经在提醒：

- 第三层适合全国风险识别
- 第二层更适合机制专题

如果后续不把这两条线彻底分开，课题很容易在解释上变得不自洽。

## 8. 截至目前最稳妥的阶段性结论

截至当前，最稳妥的课题结论可以表述为：

1. SYR4 可以支撑基于 `PWS-year` 的全国尺度 DBP 风险识别，但该层级更适合年度系统风险画像，不适合承载精细机制结论。
2. 严格样本级无法支撑多变量主建模，因此分层建模思路是必要的。
3. `facility-month` 应作为后续机制分析与专题 reduced dataset 主线保留。
4. 第三层主线上，`TOC` 是目前最可信的机制增强变量。
5. `free_chlorine` 只有弱边际增益，当前不能被稳健解释为数值本身的强机制信号。
6. `total_chlorine` 当前不适合继续进入第三层正式增量实验。
7. 当前最重要的成果不是找到一个单一“终极最优模型”，而是把第三层主线的适用边界和变量增量边界基本探清。

## 9. 后续最合理的方向

当前更合理的下一步，不是继续在第三层主线上硬加高稀疏变量，而是：

1. 固定第三层主线当前结论与解释边界。
2. 如果继续推进全国主线，优先测试覆盖率更高、结构更稳定的变量。
3. 把第二层 `facility-month` 正式接成机制专题主线。
4. 对高稀疏扩展变量，先做 readiness audit，再决定是否值得进入正式实验。

## 10. 一句话总结

到目前为止，课题最重要的阶段性收获不是“模型性能已经最终定型”，而是：

**已经较为清楚地识别出，SYR4 在第三层 `PWS-year` 上最适合做什么、不适合做什么，哪些变量值得保留在全国主线，哪些变量应该转回第二层专题机制线。**
