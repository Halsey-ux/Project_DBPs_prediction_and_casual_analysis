# V5.3 框架决策支持定义

- 更新时间：2026-04-10 10:35（Asia/Hong_Kong）
- 对应阶段：`V5.3`
- 承接阶段：`V5.2`、`V5.2b`

## 1. 文档定位

本文档不是新的建模执行报告，而是对整个 `V5` 线在框架层面的正式收束更新。

`V5.3` 当前的核心任务不再是直接进入 `TOC` 建模，而是先回答一个更基础的问题：

- 这个分层框架到底要解决什么实际问题
- 对监管者和水厂而言，应输入什么信息
- 输出结果应该具有什么含义
- 这个框架与“直接根据已有监测结果做监管汇总”有什么本质区别

## 2. `V5.3` 当前的正式主题

`V5.3` 当前正式主题应定义为：

- 基于真实监管数据的分层风险证据与决策支持框架定义

更具体地说，本轮要做的不是继续扩第二层变量链，而是正式固定：

1. 框架的现实使用对象
2. 框架的输入输出逻辑
3. 第三层和第二层在最终框架中的不可替代角色
4. 整个框架对监管者的真实价值边界

## 3. 框架要解决的实际问题

当前最稳妥的问题定义不是：

- 用监管数据去重复预测某个系统某一年 `TTHM/HAA5` 是否高风险

而应定义为：

- 在真实监管数据不完整、变量覆盖不均衡、信息密度差异显著的条件下，如何利用不同层级的信息生成受监管 `DBP` 高风险场景证据，从而支持优先级排序、重点复核和补充监测决策。

因此，框架的实际问题是：

- 风险筛查
- 优先级排序
- 高风险场景识别
- 局部补充证据整合

而不是：

- 直接替代监管者根据完整结果汇总做出的正式监管判定

## 4. 现实使用对象

### 4.1 对环保局

框架最主要服务的现实问题是：

- 哪些系统值得优先关注
- 哪些对象值得优先复核
- 哪些对象需要补充监测或加密监测
- 在信息不完整时，如何先建立风险视野

### 4.2 对水厂

框架能提供的价值是：

- 判断系统整体是否属于重点风险对象
- 识别哪些设施、哪些月份更值得优先关注
- 帮助安排有限监测和排查资源

## 5. 框架输入逻辑

### 5.1 第三层主筛查输入

第三层 `PWS-year` 主模型应优先使用覆盖较广、制度上更稳定的系统级基础数据，例如：

- 系统类型
- 水源类型
- 服务人口
- 处理摘要信息
- 结构/覆盖相关信息

其输入逻辑应尽量避免：

- 使用可直接还原同周期 `DBP` 结果的同源信息
- 把同周期目标结果或结果汇总回灌为输入特征

### 5.2 第二层局部支撑输入

第二层 `facility-month` 模块应优先使用少量高信息、但覆盖受限的局部数据，例如：

- `pH`
- `alkalinity`
- 后续若保留专题分支，再考虑 `TOC`

第二层的功能不是形成另一个广覆盖正式主模型，而是：

- 提供局部高风险场景支撑证据
- 提供更接近环境过程的补充解释

## 6. 框架输出逻辑

整个框架不应只输出一个单一标签或单一预测值。

更合理的输出是：

1. 系统级优先关注等级
2. 是否存在局部高信息支撑证据
3. 证据主要来自哪一类信息通路
4. 当前证据强度高/中/低
5. 建议动作

建议动作可以包括：

- 常规跟踪
- 优先复核
- 补充监测
- 加密监测
- 专题调查

## 7. 第三层与第二层在最终框架中的角色

### 7.1 第三层 `PWS-year`

正式定位：

- 广覆盖主筛查层

它回答的问题是：

- 哪些系统在系统-年份尺度上属于高优先级疑似高风险对象

它的价值不在于替代监管者直接计算最终结果，而在于：

- 在广覆盖范围内快速建立风险优先级

### 7.2 第二层 `facility-month`

正式定位：

- 局部场景支撑层

它回答的问题是：

- 在少量高信息对象中，是否存在更具体、更接近过程层面的局部高风险场景证据

它的价值在于：

- 为第三层主筛查结果提供补充支撑
- 为局部设施和月份定位提供帮助

### 7.3 两层关系

两层关系不应写成：

- 两个竞争性主模型

而应写成：

- 第三层负责广覆盖主筛查
- 第二层负责高信息局部支撑
- 两者共同构成一个分层风险证据框架

## 8. 与“直接做监管汇总”的区别

这个框架真正有意义的前提是：

- 它解决的不是“对已完整观测结果做重复汇总”
- 而是“在结果不完整、监测不均衡时先做风险筛查与优先级排序”

因此必须承认：

- 如果某对象的目标结果已经完整、清晰且可直接汇总，则框架对该对象的额外价值会下降
- 框架最主要的价值场景，是信息不完整、资源有限和需要提前筛查的现实监管环境

## 9. `V5.3` 对后续工作的直接影响

本轮更新后，后续工作应按以下逻辑推进：

1. 第三层继续作为广覆盖主筛查层固定
2. 第二层继续作为局部高信息支撑层固定
3. 整合层以后应输出综合风险等级、证据来源与建议动作
4. `TOC reduced dataset` 不再作为 `V5.3` 的默认主题，而是下沉为后续专题子分支

## 10. 本轮结论

`V5.3` 当前最重要的更新不是再加变量，而是正式把整个项目中的 `V5` 线定义为：

- 一个基于真实监管数据、通过不同层级模型生成并整合高风险场景证据、最终服务于优先级排序和重点复核决策的分层风险支持框架。
