## 1. 当前整体课题思路

本课题当前不应被理解为单一的 `DBP` 浓度回归模型，也不应被理解为只预测某个系统某一年是否已经处于 `TTHM/HAA5` 高风险状态的普通分类模型。

当前更稳妥的整体定位是：

- 基于美国 `SYR4` 真实监管数据，构建一个受监管消毒副产物高风险场景的分层风险证据与决策支持框架。

这个框架要解决的实际问题是：

- 在监管数据不完整、变量覆盖不均衡、信息密度差异显著的现实条件下，如何利用不同层级的信息生成受监管 `DBP` 高风险场景证据，从而支持优先级排序、重点复核和补充监测决策。

因此，当前框架的核心价值不是：

- 替代监管者对完整监测结果进行直接汇总
- 用模型重复计算已经清晰可见的监管结果
- 给出不加边界的因果结论

而是：

- 在信息不完整或监测不均衡时，先识别疑似高风险对象
- 在广覆盖系统层面形成风险优先级
- 在少量高信息设施-月份样本中提供局部场景支撑
- 为环保局或水厂提供“谁先看、哪里先查、哪些对象值得补充监测”的决策支持

## 2. 当前框架的层级分工

### 2.1 第三层：`PWS-year` 广覆盖主筛查层

第三层 `PWS-year` 是当前框架的广覆盖主筛查层。

它的主要输入应优先包括：

- 系统类型
- 水源类型
- 服务人口
- 处理摘要信息
- 结构/覆盖相关信息

它的核心输出不是正式监管判定，而是：

- 系统级疑似高风险评分
- 年度优先关注等级
- 面向环保局的广覆盖风险排序

它解决的问题是：

- 在很多系统之间，哪些对象应优先被关注、复核或补充监测。

### 2.2 第二层：`facility-month` 局部场景支撑层

第二层 `facility-month` 是当前框架的局部场景支撑层。

它的主要输入是少量高信息但覆盖受限的局部变量，例如：

- `pH`
- `alkalinity`
- 后续可能作为专题子分支评估的 `TOC`

它的核心输出不是替代第三层主筛查，而是：

- 局部设施-月份风险支撑证据
- 某些设施、某些月份是否存在更接近水质过程的补充风险信号

它解决的问题是：

- 在某个系统已经被识别为疑似高风险后，是否存在局部高信息证据支持进一步复核或加密监测。

### 2.3 后续嵌套模块：解释与因果专题层

因果分析可以作为后续嵌套模块，但不应作为当前主框架的主输出。

当前合理定位是：

- 主框架先负责风险筛查、证据整合和优先级排序
- 后续因果专题模块再针对重点对象、重点变量和更适合的子样本，评估潜在驱动因素和干预假设

因果模块不能直接替代当前主线，也不能把预测关联直接写成因果机制。

## 3. 当前框架的最终输出形式

最终框架不应只输出一个单一的“高风险/非高风险”标签。

更合理的输出应包括：

- 综合风险等级
- 系统级优先关注等级
- 是否存在局部高信息支撑
- 证据来源标签
- 证据强度
- 建议动作

建议动作可以包括：

- 常规跟踪
- 优先复核
- 补充监测
- 加密监测
- 专题调查

一个合理的输出示例是：

- 系统：`A`
- 年份：`2021`
- 系统级风险等级：高
- 局部支撑证据：有
- 重点设施：`F1`
- 重点月份：`7-8 月`
- 证据来源：系统结构风险 + 局部水质支撑
- 建议动作：优先复核，并对 `F1` 夏季月份加密监测

## 6. 接下来应该做什么

### 6.1 第一优先级：定义整合层输出规则

下一步最建议先做：

- `V5.4` 或新的框架整合任务

目标是正式定义：

- 第三层风险评分如何转成 `高/中/低` 优先级

建议输出文档：

- `Framework_Risk_Evidence_Integration_Protocol.md`
- 或 `V5_4_Risk_Evidence_Integration_Protocol.md`

### 6.2 第二优先级：决定是否启动 `TOC` 专题子分支

在整合层规则明确之后，再决定是否启动：

- `TOC reduced dataset` 专题审计

如果启动，应先做审计，不应直接建模。

建议步骤：

1. 重新检查 `TTHM + pH + alkalinity + TOC` 的第二层样本规模
2. 检查 train / validation / test 中正例数是否足够
3. 如果 split 后不稳定，则直接写成专题边界结论
4. 如果可行，再考虑受限小样本模型

### 6.3 第三优先级：设计因果分析后续模块

在主框架和整合层固定后，再设计因果模块。

建议定位为：

- 重点对象和重点变量的后续专题分析

不建议定位为：

- 主框架默认输出

建议先写设计文档，而不是直接跑因果模型。

### 6.4 第四优先级：整理论文框架总结

当整合层定义完成后，应整理一份更面向论文写作的总结文档，包括：

- 研究问题
- 数据层级
- 模型层级
- 框架输入输出
- 结果解释边界
- 对环保局和水厂的实际意义
