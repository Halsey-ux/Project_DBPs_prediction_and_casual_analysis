# V5.2b Facility-Month 后续行动计划

- 更新时间：2026-04-10 10:05（Asia/Hong_Kong）
- 对应阶段：`V5.2b`

## 1. 文档定位

本文档用于把 `V5.2` 之后的后续动作整理成一个清晰、收敛、可执行的短计划。

其核心目的是避免：

- 还没固定第二层制度边界，就继续默认扩变量

## 2. 当前阶段判断

截至 `V5.2b`，最稳妥的判断是：

- 第二层 `facility-month` 已经具备受限但真实的机制支撑价值
- 第二层当前还不具备稳定成熟的增强主链证据
- 后续推进优先级应从“继续跑模型”切换为“先固定制度和专题分支边界”

## 3. 建议的下一步顺序

### 3.1 第一优先级

固定 `V5` 当前制度表述：

- `V5` 当前不是三级样本制度
- `V5` 当前是“正式任务池 + 专题子样本池”
- 第二层当前角色是“机制支撑线为主、有限高信息增强为辅”

### 3.2 第二优先级

如需继续推进，先单独启动：

- `V5.3 TOC reduced dataset` 专题审计/设计

### 3.3 第三优先级

只有在 `V5.3` 专题仍然显示值得保留时，才继续做小范围正式实验。

## 4. 当前不建议的路线

当前不建议：

- 直接进入 `TOC` 正式建模
- 继续默认扩到 `free_chlorine`
- 在第二层中仿照第三层硬做样本等级体系
- 为了追求分数而提前切换更复杂模型

## 5. 本文档结论

`V5.2b` 之后，最合理的后续动作不是继续默认加变量，而是：

1. 先固定第二层制度和边界
2. 再把 `TOC` 改写为专题分支
3. 最后才决定是否启动真正的 `V5.3`
