# V5.2b Facility-Month 样本制度与边界总结

- 更新时间：2026-04-10 10:05（Asia/Hong_Kong）
- 对应阶段：`V5.2b`
- 承接阶段：`V5.0`、`V5.1`、`V5.2`

## 1. 文档定位

本文档不是新一轮建模实验报告，而是对 `V5` 当前阶段的样本制度、模块边界与后续推进条件进行阶段性收束。

`V5.2b` 的任务是把以下问题写清楚：

1. `V5` 当前有没有像 `V4` 一样形成正式三级样本制度
2. `V5.2` 的弱正向结果意味着什么，不意味着什么
3. 第二层下一步应如何推进，哪些动作当前不宜直接做

## 2. 当前 `V5` 的正式样本制度

### 2.1 当前没有形成 `V4` 式正式三级样本制度

截至 `V5.2b`，`V5` 还没有像 `V4` 那样被正式冻结成：

- `第一级样本`
- `第二级样本`
- `第三级样本`

这不是遗漏，而是当前第二层对象本身决定的结果：

- `V4` 的三级样本制度建立在第三层 `PWS-year` 主表及其逐步增强链上
- `V5` 当前研究对象是真正第二层 `facility-month`
- 第二层在变量覆盖上远弱于第三层全国主表，因此当前更适合使用“正式任务样本池 + 受限专题子样本池”的制度，而不是直接照搬 `V4` 的三级样本框架

### 2.2 `V5` 当前实际采用的是双层样本制度

截至目前，`V5` 已经实际形成两套正式可用口径：

#### A. 第二层正式任务样本池

定义：

- `tthm_mean_ug_l` 非缺失的真正第二层 `facility-month` 月样本

定位：

- 第二层正式 baseline 任务池
- 第二层月度高风险识别正式对象

在 `V5.1` 中，该样本池承担：

- `tthm_high_risk_month_prediction`

#### B. 第二层专题 complete-case 子样本池

定义：

- 在正式 baseline 字段完整的前提下，再要求某一专题变量链完整

当前已经正式使用的是：

- `baseline + pH + alkalinity` 子样本池

定位：

- 第二层机制支撑线专题验证子样本
- 同子样本 reference 对照池

### 2.3 `match_quality_tier` 只是审计标签，不是正式训练分层制度

`V5.0` 已经给出第二层内部的覆盖审计分层：

- `D_outcome_only`
- `C_outcome_plus_1_core`
- `B_outcome_plus_2_core`
- `A_outcome_plus_3plus_core`

但当前必须明确：

- 这套分层主要服务于信息密度审计与边界判断
- 它还没有被正式写成 `V5` 的训练样本等级制度
- 当前不能把它直接等价写成 `V4` 式的 `第一级样本 / 第二级样本 / 第三级样本`

## 3. `V5.2` 对第二层角色的实际影响

### 3.1 `V5.2` 已经证明了什么

`V5.2` 已经证明：

- 真正第二层 `facility-month` 上的 `baseline + pH + alkalinity` 不是空信号
- 在同子样本对照下，`test` 上三项核心指标均改善
- 第二层“机制支撑线”有继续保留的价值

### 3.2 `V5.2` 还没有证明什么

`V5.2` 还没有证明：

- 第二层已经形成稳定成熟的增强主链
- `baseline + pH + alkalinity` 已经具备可无缝扩展到 `TOC` 的条件
- 第二层已经值得仿照 `V4` 立即演化成完整三级样本框架

原因很明确：

- `validation PR-AUC` 没有同步改善
- 专题子样本规模仍然很小
- 第二层高信息链条在当前仍然是少量高信息月样本，而不是稳定宽覆盖模块

### 3.3 `V5.2` 之后第二层最稳妥的角色表述

截至 `V5.2b`，第二层 `facility-month` 的最稳妥角色应固定为：

- 机制支撑线为主
- 有限高信息增强为辅

不应写成：

- 已成熟的第二层正式增强主模型
- 可与第三层全国主线并列竞争的宽表模型

## 4. 当前不宜直接做的事

基于 `V5.2` 的结果，当前不宜直接进行以下动作：

1. 不宜直接把 `V5.3` 写成默认线性主链下一步
2. 不宜把 `TOC` 直接接到现有机制链后面当作自然增量实验
3. 不宜为了仿照 `V4` 而强行定义第二层三级样本制度
4. 不宜现在就进入树模型、复杂调参或更重的模型路线

这些动作都存在同一个问题：

- 它们会把一个当前仍然证据有限的第二层专题机制线，误推进成成熟增强主链

## 5. 下一步的合理推进方式

当前更合理的推进方式应是：

1. 先把第二层样本制度、模块边界和结论边界固定清楚
2. 再把 `TOC` 改写为第二层 reduced dataset 专题分支
3. 只有在专题分支任务定义清楚之后，才决定是否进入真正的 `V5.3`

## 6. 本文档结论

`V5.2b` 的核心结论是：

1. `V5` 当前尚未形成 `V4` 式正式三级样本制度。
2. `V5` 当前更适合维持“正式任务样本池 + 专题 complete-case 子样本池”的双层制度。
3. `V5.2` 足以支持保留第二层“机制支撑线”，但不足以支持直接继续扩大第二层主链。
4. 第二层下一步应先做专题分支重定义，而不是直接继续加变量。
